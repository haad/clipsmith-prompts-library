From prompts/coding/code-review-swift.md (reference):
```markdown
---
title: "Swift Code Review"
version: 1
description: "Review Swift code for correctness, safety, and idiomatic style"
variables: ["clipboard"]
---

Review this Swift code for correctness, safety, and style. Focus on:

1. **Correctness** — logic errors, off-by-one, nil handling, race conditions
2. **Safety** — force unwraps, unowned references, unchecked casts
3. **Style** — naming conventions, Swift idioms, unnecessary complexity
4. **Performance** — obvious N+1 patterns, unnecessary allocations

Code to review:

{{clipboard}}

Provide specific line-by-line feedback with suggested fixes.
```

Key format rules:
- YAML frontmatter with `title` (quoted string), `version` (integer = 1), `description` (quoted string), `variables` (array)
- Body: direct instruction + focused requirements + `{{clipboard}}` + output expectation
- All titles MUST be quoted (prevents colon truncation issues)
- No leading blank lines after frontmatter separator

- A prompt body that opens with a direct instruction, includes 2-5 focused requirements, uses `{{clipboard}}` for user input, and closes with an output expectation

Use these exact filenames, titles, and descriptions (from REQUIREMENTS.md / RESEARCH.md):

1. **explain-code.md** — Title: "Explain Code", Description: "Step-by-step explanation of how code works"
   Body: Ask AI to explain the code step by step. Cover what it does, how it does it, key decisions. Assume reader is unfamiliar with codebase.

2. **fix-bug.md** — Title: "Fix Bug", Description: "Find and fix bugs in code"
   Body: Ask AI to find and fix bugs. Check for logic errors, edge cases, type mismatches, off-by-one. Explain what was wrong and why the fix is correct.

3. **write-tests.md** — Title: "Write Tests", Description: "Generate comprehensive unit tests"
   Body: Ask AI to write unit tests. Cover happy path, edge cases, error conditions. Include setup/teardown if needed. Use descriptive test names.

4. **refactor-code.md** — Title: "Refactor Code", Description: "Improve code structure without changing behavior"
   Body: Ask AI to refactor for readability, maintainability, and simplicity. Preserve external behavior. Explain each change and why it improves the code.

5. **add-error-handling.md** — Title: "Add Error Handling", Description: "Add proper error handling to code"
   Body: Ask AI to add error handling. Cover input validation, nil/null checks, try-catch for operations that can fail, meaningful error messages. Keep error handling proportional to risk.

6. **convert-to-async.md** — Title: "Convert to Async/Await", Description: "Modernize callback/completion handler code"
   Body: Ask AI to convert callbacks/completion handlers to async/await. Preserve behavior, handle errors with do-catch, maintain cancellation support where applicable.

7. **optimize-performance.md** — Title: "Optimize Performance", Description: "Identify and fix performance issues"
   Body: Ask AI to identify performance bottlenecks. Check for unnecessary allocations, N+1 patterns, redundant computation, blocking operations. Suggest specific fixes with expected improvement.
