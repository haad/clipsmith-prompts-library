  <name>Task 1: Add structural validation to build.sh</name>
  <files>scripts/build.sh</files>
  <action>
Add a structural validation step to the END of `scripts/build.sh`, after the prompts.json file is written and the count is printed. This step uses `jq -e` to verify the assembled output:

1. Check that `prompts.json` is valid JSON with the expected top-level shape:
   - `.version` exists and is a positive integer
   - `.prompts` is a non-empty array
   - Every prompt object has all 5 required fields (`id`, `title`, `category`, `version`, `content`)
   - No prompt has an empty `id`, `title`, or `content`

2. Use `jq -e` which exits non-zero if the expression evaluates to false/null. Example:
   ```bash
   if ! jq -e '
     (.version | type == "number" and . > 0) and
     (.prompts | type == "array" and length > 0) and
     (.prompts | all(has("id", "title", "category", "version", "content"))) and
     (.prompts | all(.id != "" and .title != "" and .content != ""))
   ' "$OUTPUT" > /dev/null; then
       echo "ERROR: prompts.json failed structural validation" >&2
       exit 1
   fi
   echo "Validation passed: prompts.json structure is valid"
   ```

3. This goes AFTER the existing "Built $OUTPUT" echo line (so the count message still prints before validation).

Important: Do NOT add any npm/node dependencies. Do NOT attempt JSON Schema validation with jq (jq 1.6 does not support it). The structural jq check is sufficient per the locked decision.

Important: Preserve all existing build.sh code exactly as-is. Only APPEND the validation block.

Create `.github/workflows/deploy.yml` with the following exact structure:

- `contents: write` (required for GITHUB_TOKEN to push to gh-pages)

**Job: `build-and-deploy`** on `ubuntu-latest`:

Step 1 — Checkout:
```yaml
- name: Checkout
  uses: actions/checkout@v4
```

Step 2 — Build and validate:
```yaml
- name: Build and validate
  run: bash scripts/build.sh
```
(build.sh handles compilation + validation in one step per locked decision)

Step 3 — Prepare deploy staging (CICD-04):
```yaml
- name: Prepare deploy staging
  run: |
    mkdir -p _deploy
    cp prompts.json _deploy/prompts.json
    cp index.html _deploy/index.html
```

Step 4 — Deploy to gh-pages (CICD-03):
```yaml
- name: Deploy to gh-pages
  uses: peaceiris/actions-gh-pages@v4
  with:
    github_token: ${{ secrets.GITHUB_TOKEN }}
    publish_dir: ./_deploy
    force_orphan: true
```

Use `force_orphan: true` to keep gh-pages branch clean (single commit, no history accumulation).
