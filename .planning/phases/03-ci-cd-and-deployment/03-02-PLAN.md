Create `index.html` at the repository root as a single self-contained file with all CSS and JS inline. Zero external dependencies.

**Page structure:**

1. **Header area:**
   - Title: "Flycut Prompt Library" (or similar)
   - Subtitle mentioning the prompt count and purpose
   - Dark/light mode toggle button in the top-right corner

2. **Category filter tabs (locked decision):**
   - Horizontal tab bar: All (23) | Coding (8) | Writing (6) | Analysis (5) | Creative (4)
   - Counts should be dynamically calculated from the loaded data, not hardcoded
   - Active tab visually highlighted
   - Clicking a tab filters the card grid to that category

3. **Search/filter bar (locked decision):**
   - Text input that filters prompts by title as the user types
   - Filtering is instant (no debounce needed for 23 items)
   - Works in combination with category tabs (search within selected category)
   - Placeholder text like "Search prompts..."

4. **Prompt card grid (locked decision):**
   - Responsive grid: 1 column on mobile, 2 on tablet, 3 on desktop
   - Each card contains:
     - Title (prominent)
     - Category badge (colored pill/tag)
     - Full prompt content visible (NOT collapsed — locked decision)
     - Copy-to-clipboard button
   - Content should preserve whitespace/newlines from the prompt body
   - Cards should have subtle shadows and rounded corners

5. **Copy-to-clipboard (locked decision):**
   - Use `navigator.clipboard.writeText()` (modern Clipboard API)
   - Show "Copied!" feedback for ~2 seconds, then revert to original label
   - Copy the raw content text (not HTML)

6. **Dark/light mode (locked decision):**
   - CSS custom properties for theming on `:root` (light) and `[data-theme="dark"]` (dark)
   - Apply saved theme from `localStorage` BEFORE DOM renders to prevent flash of wrong theme
   - Check `window.matchMedia('(prefers-color-scheme: dark)')` for system preference on first visit
   - Toggle button persists choice to `localStorage.setItem('theme', ...)`
   - Theme toggle should use a sun/moon icon or text indicator

7. **Data loading:**
   - `fetch('./prompts.json')` — relative URL (works on gh-pages and local server)
   - Parse JSON, extract `.prompts` array
   - Show a loading state while fetching
   - Show an error message if fetch fails

8. **Styling inspiration (locked decision):**
   - Modern, clean aesthetic inspired by prompts.chat and Goose prompt library
   - System font stack (no external font loading)
   - Smooth transitions on theme toggle and hover states
   - Category badge colors: different color per category for visual distinction
   - The page should feel polished — "designed as a foundation to build upon later"

**Technical constraints:**
- Single file, all CSS in `<style>`, all JS in `<script>`
- No external CDN links, no framework imports
- Must work when served over HTTPS (GitHub Pages)
- For local testing: `python3 -m http.server 8080` then visit `localhost:8080`
- Content with `{{variable}}` placeholders should display them as literal text (not interpreted)
- Use `white-space: pre-wrap` or similar for content to preserve formatting

**What NOT to include (deferred ideas):**
- No full-text search within prompt content body (deferred)
- No expand/collapse on cards (deferred — full content always visible)
- No tag filtering or multi-criteria filtering (deferred)
- No download/export functionality
