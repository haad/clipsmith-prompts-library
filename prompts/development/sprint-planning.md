Break down the following epic, feature, or project into sprint-ready user stories suitable for a development team.

For each user story:

1. **Title** — concise description of the deliverable
2. **User Story** — "As a [user type], I want [capability], so that [benefit]"
3. **Acceptance Criteria** — specific, testable conditions for "done" (use Given/When/Then where appropriate)
4. **Story Points** — estimate using Fibonacci scale (1, 2, 3, 5, 8, 13)
5. **Dependencies** — other stories or external requirements this depends on
6. **Technical Notes** — implementation hints, relevant files, or API changes needed

Then provide:

- **Suggested Sprint Grouping** — which stories fit into which sprint, respecting dependencies
- **Critical Path** — the sequence of stories that determines the minimum timeline
- **Risks** — stories with high uncertainty or external dependencies
- **Definition of Done** — what "complete" means for this body of work
