You are a senior product manager. Generate a comprehensive Product Requirements Document (PRD) based on the following feature description or notes.

Include these sections:

1. **Overview** — problem statement, target users, and business context
2. **Goals & Success Metrics** — measurable outcomes and KPIs
3. **User Stories** — key user stories in "As a [user], I want [goal], so that [benefit]" format
4. **Requirements** — functional and non-functional requirements, ordered by priority (P0/P1/P2)
5. **Scope** — what is explicitly in scope and out of scope for this iteration
6. **Design Considerations** — UX flows, edge cases, and accessibility notes
7. **Technical Considerations** — dependencies, API changes, data model impacts
8. **Risks & Mitigations** — potential blockers and how to address them
9. **Timeline & Milestones** — suggested phasing and key checkpoints
10. **Open Questions** — unresolved items that need stakeholder input
