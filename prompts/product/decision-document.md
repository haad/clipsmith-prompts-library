Create a structured decision document based on the following context. This document should help a team align on a decision quickly and confidently.

Structure:

1. **Decision Title** — a clear one-line statement of what needs to be decided
2. **Context** — background information, why this decision is needed now
3. **Options** — list each viable option with:
   - Description
   - Pros (3-5 bullet points)
   - Cons (3-5 bullet points)
   - Estimated effort and timeline
   - Risks
4. **Evaluation Criteria** — the factors that matter most for this decision (e.g. speed to market, cost, user experience, technical debt)
5. **Recommendation** — which option to go with and why, tied back to the evaluation criteria
6. **What We're Trading Off** — be explicit about what we're giving up
7. **Reversibility** — how easy is it to change course if this turns out wrong?
8. **Next Steps** — concrete actions if the recommendation is approved
